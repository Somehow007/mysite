## 二、架构决策

### 2.1 Elasticsearch：保留并升级 ✅ 已完成

**决策：保留 Elasticsearch，但进行重大升级。**

理由：
- 博客搜索是核心功能，ES 的全文检索能力远超 MySQL LIKE
- 当前 ES 已部署（Docker），基础设施已就绪
- 但当前实现质量极差（standard 分词器不支持中文、手动同步），需要全面升级

升级要点：
1. **安装 IK 分词器**：ES 7.17.21 对应 `elasticsearch-analysis-ik-7.17.21` ✅ 已完成
2. **重建索引映射**：`title`/`content` 使用 `ik_max_word` + `ik_smart` 双分析器 ✅ 已完成
3. **引入 Canal 或 MQ 异步同步**：替代 Service 层手动同步，保证数据一致性 ❌ 未完成
4. **多字段联合搜索**：支持标题+内容+作者同时搜索，加权排序 ✅ 已完成

### 2.2 后端重构：中度重构 ✅ 已完成

**决策：进行中度重构，不推倒重来。**

理由：
- 当前代码结构基本合理（Controller→Service→Mapper 分层清晰）
- 但存在多个影响功能正确性的 Bug 和设计缺陷
- 需要补充分类/标签体系才能支撑现代博客前端

重构范围：

| 模块 | 动作 | 优先级 | 状态 |
|------|------|--------|------|
| 认证体系 | Session → JWT | P0 | ✅ 已完成 |
| 数据库 | 新增分类表、标签表、文章-标签关联表 | P0 | ✅ 已完成 |
| ES | IK 分词 + 索引重建 + Canal 同步 | P0 | ⚠️ IK 分词已完成，Canal 未完成 |
| ArticleDO | 新增 coverImage、category_id 字段 | P1 | ✅ 已完成 |
| 分页 | 统一分页参数，去除硬编码 | P1 | ✅ 已完成 |
| Bug 修复 | updateUser 邮箱字段、CORS 配置 | P1 | ✅ 已完成 |
| API 版本 | 添加 `/v1/` 前缀 | P2 | ✅ 已完成 |

### 2.3 前端：完全重写 ✅ 已完成

**决策：删除全部前端代码，从零重建。**

理由：
- 当前 CSS 架构（35k 单文件）无法维护和演进
- Attila 主题风格与"现代风、高级感"目标完全不符
- 残留大量脚手架代码
- 无组件库/CSS 框架，迭代成本极高

**实际执行情况**：
- ✅ 已完全删除旧代码
- ✅ 使用 TailwindCSS 4 重建
- ✅ 实现了极简设计风格
- ✅ 所有核心页面已完成
