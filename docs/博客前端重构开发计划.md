## 二、架构决策

### 2.1 Elasticsearch：保留并升级 ✅

**决策：保留 Elasticsearch，但进行重大升级。**

理由：
- 博客搜索是核心功能，ES 的全文检索能力远超 MySQL LIKE
- 当前 ES 已部署（Docker），基础设施已就绪
- 但当前实现质量极差（standard 分词器不支持中文、手动同步），需要全面升级

升级要点：
1. **安装 IK 分词器**：ES 7.17.21 对应 `elasticsearch-analysis-ik-7.17.21`
2. **重建索引映射**：`title`/`content` 使用 `ik_max_word` + `ik_smart` 双分析器
3. **引入 Canal 或 MQ 异步同步**：替代 Service 层手动同步，保证数据一致性
4. **多字段联合搜索**：支持标题+内容+作者同时搜索，加权排序

### 2.2 后端重构：中度重构 ✅

**决策：进行中度重构，不推倒重来。**

理由：
- 当前代码结构基本合理（Controller→Service→Mapper 分层清晰）
- 但存在多个影响功能正确性的 Bug 和设计缺陷
- 需要补充分类/标签体系才能支撑现代博客前端

重构范围：

| 模块 | 动作 | 优先级 |
|------|------|--------|
| 认证体系 | Session → JWT | P0 |
| 数据库 | 新增分类表、标签表、文章-标签关联表 | P0 |
| ES | IK 分词 + 索引重建 + Canal 同步 | P0 |
| ArticleDO | 新增 coverImage、category_id 字段 | P1 |
| 分页 | 统一分页参数，去除硬编码 | P1 |
| Bug 修复 | updateUser 邮箱字段、CORS 配置 | P1 |
| API 版本 | 添加 `/v1/` 前缀 | P2 |

### 2.3 前端：完全重写 ✅

**决策：删除全部前端代码，从零重建。**

理由：
- 当前 CSS 架构（35k 单文件）无法维护和演进
- Attila 主题风格与"现代风、高级感"目标完全不符
- 残留大量脚手架代码
- 无组件库/CSS 框架，迭代成本极高

### 4.2 色彩体系

```
Light Mode:
  背景:    #FAFAFA (主) / #FFFFFF (卡片) / #F5F5F5 (代码块)
  文字:    #18181B (主标题) / #3F3F46 (正文) / #A1A1AA (次要)
  强调:    #18181B (黑，非彩色强调 — 高级感核心)

Dark Mode:
  背景:    #09090B (主) / #18181B (卡片) / #27272A (代码块)
  文字:    #FAFAFA (主标题) / #D4D4D8 (正文) / #71717A (次要)
  强调:    #FAFAFA (白，暗色下的强调)
```

> **设计原则**：不使用彩色强调色（如蓝/紫），纯黑白灰营造高级感。交互元素（链接、按钮）使用中性色 hover 变化。

### 4.3 字体

```css
--font-sans: 'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
--font-mono: 'JetBrains Mono', 'Fira Code', monospace;
```

- 标题/导航：Inter（无衬线，几何感强）
- 文章正文：Noto Serif SC（中文衬线，阅读舒适）
- 代码：JetBrains Mono

### 6.1 数据库变更

```sql
-- 新增：分类表
CREATE TABLE IF NOT EXISTS `t_category` (
    `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    `name` VARCHAR(50) NOT NULL COMMENT '分类名称',
    `slug` VARCHAR(50) NOT NULL COMMENT '分类别名（URL友好）',
    `description` VARCHAR(200) DEFAULT NULL COMMENT '分类描述',
    `sort_order` INT DEFAULT 0 COMMENT '排序',
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    `del_flag` TINYINT NOT NULL DEFAULT 0,
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_slug` (`slug`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='分类表';

-- 新增：标签表
CREATE TABLE IF NOT EXISTS `t_tag` (
    `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    `name` VARCHAR(50) NOT NULL COMMENT '标签名称',
    `slug` VARCHAR(50) NOT NULL COMMENT '标签别名',
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    `del_flag` TINYINT NOT NULL DEFAULT 0,
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_slug` (`slug`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='标签表';

-- 新增：文章-标签关联表
CREATE TABLE IF NOT EXISTS `t_article_tag` (
    `id` BIGINT NOT NULL AUTO_INCREMENT,
    `article_id` BIGINT NOT NULL COMMENT '文章ID',
    `tag_id` BIGINT NOT NULL COMMENT '标签ID',
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `del_flag` TINYINT NOT NULL DEFAULT 0,
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_article_tag` (`article_id`, `tag_id`),
    KEY `idx_tag_id` (`tag_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='文章标签关联表';

-- 修改：文章表新增字段
ALTER TABLE `t_article`
    ADD COLUMN `cover_image` VARCHAR(500) DEFAULT NULL COMMENT '封面图片URL' AFTER `summary`,
    ADD COLUMN `category_id` BIGINT DEFAULT NULL COMMENT '分类ID' AFTER `cover_image`,
    ADD KEY `idx_category_id` (`category_id`);
```

## 附录 A：后端 API 设计（新增/变更）

### 认证相关

```
POST   /v1/auth/login          # JWT 登录，返回 accessToken + refreshToken
POST   /v1/auth/logout         # 注销
POST   /v1/auth/refresh        # 刷新 Token
POST   /v1/auth/register       # 注册
GET    /v1/auth/me             # 获取当前用户信息
```

### 文章相关

```
GET    /v1/articles            # 分页获取文章列表（支持 keyword/category/tag 筛选）
GET    /v1/articles/:id        # 获取文章详情（含分类、标签）
POST   /v1/articles            # 创建文章
PUT    /v1/articles/:id        # 更新文章
DELETE /v1/articles/:id        # 删除文章
POST   /v1/articles/:id/favorite  # 收藏/取消收藏
GET    /v1/articles/favorites  # 获取收藏列表
GET    /v1/articles/archive    # 归档列表（按年月分组）
```

### 分类相关

```
GET    /v1/categories          # 获取所有分类
GET    /v1/categories/:slug    # 获取分类详情 + 该分类下的文章
POST   /v1/categories          # 创建分类
PUT    /v1/categories/:id      # 更新分类
DELETE /v1/categories/:id      # 删除分类
```

### 标签相关

```
GET    /v1/tags                # 获取所有标签
GET    /v1/tags/:slug          # 获取标签详情 + 该标签下的文章
POST   /v1/tags                # 创建标签
DELETE /v1/tags/:id            # 删除标签
```

### 搜索

```
GET    /v1/search?keyword=xxx  # 全文搜索（标题+内容+作者，ES 驱动）
```
