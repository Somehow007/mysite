## 1. 资源需求分析

### 1.1 项目技术栈

| 组件 | 技术选型 | 版本 |
|------|---------|------|
| 前端框架 | Vue 3 + TypeScript | 3.5.x |
| 构建工具 | Vite | 7.x |
| UI框架 | Tailwind CSS | 4.x |
| 后端框架 | Spring Boot | 3.0.7 |
| Java版本 | OpenJDK | 17 |
| 数据库 | MySQL | 8.0 |
| 缓存 | Redis | 7.x |
| 搜索引擎 | Elasticsearch | 7.17.21 |
| 评论系统 | Artalk | latest |
| Web服务器 | Nginx | 1.24+ |
| 容器化 | Docker | 24+ |

### 1.2 内存需求评估

| 服务 | 默认内存占用 | 最小内存配置 | 推荐内存配置 |
|------|------------|------------|------------|
| MySQL 8.0 | 400-800MB | 256MB | 512MB |
| Redis | 50-100MB | 50MB | 100MB |
| Elasticsearch | 1-2GB | 512MB | 1GB |
| Spring Boot | 512MB-1GB | 256MB | 512MB |
| Nginx | 10-50MB | 10MB | 50MB |
| Artalk | 30-50MB | 30MB | 50MB |
| 操作系统 | 300-500MB | 300MB | 500MB |
| **总计** | **2.3-4.4GB** | **1.4GB** | **2.7GB** |

### 1.3 可行性评估

#### ⚠️ 关键问题

**内存严重不足**：
- 服务器总内存：2GB
- 最小运行需求：1.4GB（极度优化后）
- 推荐运行需求：2.7GB
- **结论**：在2GB内存下全栈部署极其困难，存在以下风险：
  - 频繁OOM（内存溢出）导致服务崩溃
  - 系统swap频繁，性能严重下降
  - 无法应对流量峰值
  - 数据库和ES性能受限

#### 💡 核心瓶颈

1. **Elasticsearch**：ES是最消耗内存的组件，至少需要512MB堆内存
2. **MySQL**：InnoDB缓冲池需要足够内存保证性能
3. **Spring Boot**：JVM需要足够的堆内存运行
4. **并发能力**：内存不足会严重影响并发处理能力

## 2. 部署方案选择

### 2.1 方案对比

| 方案 | 内存占用 | 成本 | 性能 | 推荐度 | 适用场景 |
|------|---------|------|------|--------|---------|
| 方案一：全栈部署 | ~1.8GB | 低 | 差 | ⭐⭐ | 开发测试 |
| 方案二：混合部署 | ~0.8GB | 中 | 优 | ⭐⭐⭐⭐⭐ | 生产环境 |
| 方案三：轻量级部署 | ~1.2GB | 低 | 中 | ⭐⭐⭐ | 预算有限 |

### 2.2 推荐方案

**强烈推荐：方案二（混合部署）**

**理由**：
1. 将数据库和缓存迁移到云服务，释放服务器内存
2. 可选将ES迁移到云服务或使用数据库全文搜索替代
3. 服务器只运行应用层服务，性能更稳定
4. 云数据库有自动备份、高可用等企业级特性

**成本估算**（以阿里云为例）：
- 云数据库MySQL（基础版1核1G）：约 ¥50-80/月
- 云Redis（256MB）：约 ¥30-50/月
- 云Elasticsearch（可选）：约 ¥100-200/月
- **总计**：约 ¥80-330/月

### 3.3 系统优化

#### 增加Swap空间

```bash
# 创建2GB swap文件
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# 永久生效
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# 优化swap使用策略
sudo sysctl vm.swappiness=10
echo 'vm.swappiness=10' | sudo tee -a /etc/sysctl.conf
```

#### 系统参数优化

```bash
# /etc/sysctl.conf
vm.swappiness = 10
vm.overcommit_memory = 1
vm.max_map_count = 262144
net.core.somaxconn = 1024
net.ipv4.tcp_max_syn_backlog = 1024
net.ipv4.tcp_tw_reuse = 1
net.ipv4.tcp_fin_timeout = 30
```

### 3.4 风险提示

1. **OOM风险高**：内存接近极限，流量峰值可能导致服务崩溃
2. **性能差**：频繁GC和swap导致响应慢
3. **不稳定**：任何一个服务内存泄漏都会影响整体
4. **无法扩展**：没有资源应对业务增长

#### 方案B：移除ES，使用数据库搜索

**优点**：
- 节省ES内存（512MB+）
- 节省云ES费用（¥100-200/月）
- 简化架构

**缺点**：
- 搜索性能下降
- 搜索功能受限

**实现方案**：

```java
// 使用MySQL全文索引替代ES
// 1. 创建全文索引
ALTER TABLE t_article ADD FULLTEXT INDEX ft_title_content (title, content) WITH PARSER ngram;

// 2. 修改搜索实现
@Service
public class ArticleSearchService {
    
    @Autowired
    private ArticleMapper articleMapper;
    
    public List<ArticleDO> search(String keyword, int page, int size) {
        return articleMapper.searchByFulltext(keyword, (page - 1) * size, size);
    }
}
```

```xml
<!-- ArticleMapper.xml -->
<select id="searchByFulltext" resultType="ArticleDO">
    SELECT * FROM t_article
    WHERE del_flag = 0 
      AND published = 1
      AND MATCH(title, content) AGAINST(#{keyword} IN NATURAL LANGUAGE MODE)
    ORDER BY create_time DESC
    LIMIT #{offset}, #{size}
</select>
```

#### 启动脚本

```bash
#!/bin/bash
# /opt/mysite/start.sh

APP_NAME="mysite"
APP_JAR="/opt/mysite/mysite.jar"
LOG_DIR="/var/log/mysite"
PID_FILE="/var/run/mysite.pid"

# JVM参数
JAVA_OPTS="-Xms512m -Xmx768m"
JAVA_OPTS="$JAVA_OPTS -XX:+UseG1GC"
JAVA_OPTS="$JAVA_OPTS -XX:MaxGCPauseMillis=200"
JAVA_OPTS="$JAVA_OPTS -XX:+HeapDumpOnOutOfMemoryError"
JAVA_OPTS="$JAVA_OPTS -XX:HeapDumpPath=$LOG_DIR/heapdump.hprof"
JAVA_OPTS="$JAVA_OPTS -Djava.security.egd=file:/dev/./urandom"
JAVA_OPTS="$JAVA_OPTS -Dspring.profiles.active=production"

# 环境变量
export DB_HOST="rds-xxx.mysql.rds.aliyuncs.com"
export DB_PORT="3306"
export DB_USER="mysite"
export DB_PASSWORD="your-db-password"
export REDIS_HOST="r-xxx.redis.rds.aliyuncs.com"
export REDIS_PORT="6379"
export REDIS_PASSWORD="your-redis-password"
export JWT_SECRET="your-production-jwt-secret-key"

# 创建日志目录
mkdir -p $LOG_DIR

# 启动应用
start() {
    if [ -f "$PID_FILE" ]; then
        PID=$(cat $PID_FILE)
        if ps -p $PID > /dev/null 2>&1; then
            echo "$APP_NAME is already running (PID: $PID)"
            return 1
        fi
    fi
    
    echo "Starting $APP_NAME..."
    nohup java $JAVA_OPTS -jar $APP_JAR > $LOG_DIR/console.log 2>&1 &
    echo $! > $PID_FILE
    echo "$APP_NAME started (PID: $(cat $PID_FILE))"
}

# 停止应用
stop() {
    if [ -f "$PID_FILE" ]; then
        PID=$(cat $PID_FILE)
        if ps -p $PID > /dev/null 2>&1; then
            echo "Stopping $APP_NAME (PID: $PID)..."
            kill $PID
            sleep 5
            if ps -p $PID > /dev/null 2>&1; then
                kill -9 $PID
            fi
            rm -f $PID_FILE
            echo "$APP_NAME stopped"
        else
            echo "$APP_NAME is not running"
            rm -f $PID_FILE
        fi
    else
        echo "$APP_NAME is not running"
    fi
}

# 重启应用
restart() {
    stop
    sleep 2
    start
}

# 查看状态
status() {
    if [ -f "$PID_FILE" ]; then
        PID=$(cat $PID_FILE)
        if ps -p $PID > /dev/null 2>&1; then
            echo "$APP_NAME is running (PID: $PID)"
        else
            echo "$APP_NAME is not running"
        fi
    else
        echo "$APP_NAME is not running"
    fi
}

case "$1" in
    start)
        start
        ;;
    stop)
        stop
        ;;
    restart)
        restart
        ;;
    status)
        status
        ;;
    *)
        echo "Usage: $0 {start|stop|restart|status}"
        exit 1
        ;;
esac
```

#!/bin/bash
# /opt/mysite/monitor.sh

LOG_FILE="/var/log/mysite/monitor.log"
ALERT_EMAIL="admin@your-domain.com"

# 检查内存使用
check_memory() {
    local mem_usage=$(free | awk '/Mem/{printf("%.0f"), $3/$2*100}')
    if [ $mem_usage -gt 90 ]; then
        echo "[$(date)] WARNING: Memory usage is ${mem_usage}%" >> $LOG_FILE
        # 发送告警邮件或通知
    fi
}

# 检查磁盘使用
check_disk() {
    local disk_usage=$(df -h / | awk 'NR==2{print $5}' | sed 's/%//')
    if [ $disk_usage -gt 85 ]; then
        echo "[$(date)] WARNING: Disk usage is ${disk_usage}%" >> $LOG_FILE
    fi
}

# 检查服务状态
check_services() {
    services=("mysite" "nginx" "mysql" "redis-server")
    for service in "${services[@]}"; do
        if ! systemctl is-active --quiet $service; then
            echo "[$(date)] ERROR: $service is not running" >> $LOG_FILE
            # 尝试重启服务
            sudo systemctl restart $service
        fi
    done
}

# 执行检查
check_memory
check_disk
check_services

### 10.1 最终推荐方案

**理由**：
1. ✅ 性能最优：服务器资源充足，应用运行稳定

### 10.2 实施建议

1. **分阶段部署**
   - 第一阶段：部署基础服务（Nginx + Spring Boot + Artalk）
   - 第二阶段：迁移数据库到云服务
   - 第三阶段：迁移Redis到云服务
   - 第四阶段：评估是否需要Elasticsearch

2. **监控先行**
   - 部署前配置好监控告警
   - 定期检查系统资源使用情况

   - 定期分析性能瓶颈
   - 优化数据库查询和索引
