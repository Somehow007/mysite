# MySite 图片管理功能开发方案

## 项目现状

| 维度 | 现状 |
|------|------|
| 后端 | Spring Boot 3.0.7 + Java 17 + MyBatis-Plus + MySQL + Redis + Spring Security (JWT) |
| 前端 | Vue 3 + TypeScript + Vite 7 + Tailwind CSS 4 + Pinia + Axios |
| 部署 | Docker Compose + Nginx 反向代理，后端端口 8081，Nginx 端口 8080 |
| 编辑器 | 自研 MarkdownEditor，已集成图片上传按钮、拖拽上传、粘贴上传、URL上传 |

## 1. 功能需求

### 1.1 本地上传

- 工具栏按钮点击上传
- 快捷键 `Cmd/Ctrl+Shift+I` 触发上传
- 拖拽图片到编辑器上传
- 粘贴剪贴板图片上传

### 1.2 URL上传

- 输入图片 URL，后端拉取并存储到服务器
- 验证 URL 可达性和 Content-Type
- SSRF 防护

### 1.3 图片管理

- 图片删除（软删除）
- 图片信息记录（文件名、大小、尺寸、上传时间）

## 2. 技术方案

### 2.1 存储：云服务器本地

个人博客场景下，云服务器本地存储是最优选择：零成本、低延迟、实现简单、Nginx 静态文件服务性能优异。

**存储路径结构：**
```
/data/mysite/uploads/images/
  └── 2026/
      └── 05/
          ├── 18_a1b2c3d4.jpg
          └── 19_e5f6g7h8.png
```

### 2.2 架构

```
浏览器 (MarkdownEditor → image.ts API → Axios)
    │
    │ POST /v1/images/upload (multipart/form-data)
    │ POST /v1/images/upload-url (application/json)
    │
    ▼
Nginx :8080
    /v1/* ──proxy_pass──→ Spring Boot :8081
    /uploads/* ──alias──→ /data/mysite/uploads/ (静态文件)
    │
    ▼
Spring Boot :8081
    ImageController → ImageService → ImageMapper
                         ↓
                   文件系统 + MySQL t_image 表
```

## 3. 关键实现细节

### 3.1 Multipart 请求处理（核心要点）

**问题背景：** Axios 实例如果全局设置了 `Content-Type: application/json`，发送 FormData 时该默认头会覆盖 Axios 自动检测 `multipart/form-data` 的行为，导致后端报 `MultipartException: Current request is not a multipart request`。

**解决方案：**

1. **Axios 实例不设置全局 Content-Type**：Axios 会根据请求数据类型自动设置正确的 Content-Type
   - 普通对象 → `application/json`（自动）
   - FormData → `multipart/form-data; boundary=...`（自动）

2. **请求拦截器中删除 FormData 请求的 Content-Type**：双重保险，确保浏览器自动生成带 boundary 的头

```typescript
// client.ts
const apiClient = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || '',
  timeout: 10000,
  // 不设置全局 Content-Type
})

apiClient.interceptors.request.use((config) => {
  // FormData 请求删除 Content-Type，让浏览器自动设置
  if (config.data instanceof FormData) {
    delete config.headers['Content-Type']
  }
  return config
})
```

3. **后端接口显式声明 consumes**：

```java
@PostMapping(value = "/v1/images/upload", consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
public Result<ImageUploadRespDTO> uploadImage(@RequestParam("file") MultipartFile file) {
    // ...
}
```

4. **GlobalExceptionHandler 添加 MultipartException 处理**：

```java
@ExceptionHandler(value = MultipartException.class)
public Result multipartExceptionHandler(HttpServletRequest request, MultipartException ex) {
    return Results.failure(ErrorCode.IMAGE_UPLOAD_FAILED.code(), "请使用multipart/form-data格式上传文件");
}
```

### 3.2 文件存储

- 使用 `Paths.get(basePath).toAbsolutePath()` 获取绝对路径
- 使用 `Files.copy(inputStream, filePath)` 替代 `file.transferTo()`，避免相对路径问题
- 按年/月自动创建子目录

### 3.3 安全策略

| 层级 | 措施 |
|------|------|
| 前端 | `<input accept="image/...">` 限制文件选择 |
| 后端-Content-Type | 校验 MIME 类型白名单 |
| 后端-魔数 | 读取文件头字节校验（JPEG: FFD8FF, PNG: 89504E47, GIF: 47494638） |
| 后端-大小 | Spring `max-file-size: 10MB` + 业务层 5MB 校验 |
| 后端-SSRF | URL上传时过滤内网 IP（127.0.0.1, 10.x, 172.16-31.x, 192.168.x, localhost） |
| 后端-限流 | Redis 每用户每分钟 10 次上传限制 |
| 后端-文件名 | 服务端生成随机文件名，不使用原始文件名 |

### 3.4 配置项

**application.yaml：**
```yaml
spring:
  servlet:
    multipart:
      enabled: true
      max-file-size: 10MB
      max-request-size: 20MB

image:
  upload:
    base-path: ./uploads/images
    max-file-size: 5MB
    allowed-types: image/jpeg,image/png,image/gif,image/webp,image/svg+xml
    url-path: /uploads/images/
    max-uploads-per-minute: 10
    url-fetch-connect-timeout: 5000
    url-fetch-read-timeout: 15000
```

**ImageUploadConfig：**
- `maxFileSize` 使用 `DataSize` 类型 + `@DataSizeUnit(DataUnit.MEGABYTES)` 注解，避免 `5MB` 字符串绑定到 `long` 类型报错

## 5. 踩坑记录

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| `MultipartException: Current request is not a multipart request` | Axios 全局 `Content-Type: application/json` 覆盖了 FormData 的自动 multipart 头 | 移除全局 Content-Type，拦截器中删除 FormData 请求的 Content-Type |
| `Failed to bind properties: For input string: "5MB"` | `ImageUploadConfig.maxFileSize` 类型为 `long`，无法接收 `5MB` 字符串 | 改为 `DataSize` 类型 + `@DataSizeUnit` 注解 |
| `FileNotFoundException` 保存图片失败 | `file.transferTo()` 使用相对路径 | 改用 `Files.copy(file.getInputStream(), filePath)` + 绝对路径 |
| `MaxUploadSizeExceededException` 返回系统错误 | 未在 GlobalExceptionHandler 中处理该异常 | 添加专门异常处理器，返回业务错误码 |

## 6. 部署

### Nginx 配置
```nginx
location /uploads/ {
    alias /data/mysite/uploads/;
    expires 1y;
    add_header Cache-Control "public, immutable";
    add_header X-Content-Type-Options "nosniff";
    access_log off;
}
```

### Docker Volume
```yaml
services:
  mysite:
    volumes:
      - ./uploads:/data/mysite/uploads
```
